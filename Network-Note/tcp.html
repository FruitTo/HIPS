Source Port 16 bits
ระบุหมายเลข port ของแอพพลิเคชันส่ง มีค่าตั้งแต่ 0-65535 โดยแบ่งออกเป็น 3 ช่วง:
ช่วง Port Numbers:
• Well-known ports (0-1023): FTP(21), SSH(22), HTTP(80), HTTPS(443)
• Registered ports (1024-49151): แอพพลิเคชันเฉพาะ
• Dynamic/Private ports (49152-65535): ใช้สำหรับ ephemeral connections
ตัวอย่าง: 0x0050 (80), 0x01BB (443), 0xC000 (49152)
Destination Port 16 bits
ระบุหมายเลข port ของแอพพลิเคชันปลายทาง ใช้ร่วมกับ IP address เพื่อระบุ socket endpoint
การทำงาน:
Client มักใช้ dynamic port เป็น source และ well-known port เป็น destination
Server ฟังที่ well-known port และตอบกลับไปยัง client's source port
Client: src=49152, dst=80 | Server: src=80, dst=49152
Sequence Number 32 bits
ระบุลำดับของ byte แรกในส่วนข้อมูลของ segment นี้ เริ่มต้นด้วยค่าสุ่มใน SYN packet เพื่อความปลอดภัย
การนับ Sequence:
• เริ่มต้น: Initial Sequence Number (ISN) สุ่ม
• การเพิ่ม: ISN + จำนวน bytes ที่ส่งไปแล้ว
• SYN/FIN: นับเป็น 1 byte แม้ไม่มีข้อมูล
• Wrap around: กลับเป็น 0 เมื่อเกิน 2^32
SYN: seq=1000 | Data: seq=1001 (1KB) | Next: seq=2001
Acknowledgment Number 32 bits
ระบุ sequence number ถัดไปที่คาดหวังจะได้รับ มีความหมายเฉพาะเมื่อ ACK flag เป็น 1
Cumulative Acknowledgment:
• ยืนยันการรับข้อมูลทั้งหมดก่อนหน้า sequence number นี้
• ไม่ใช่ selective acknowledgment (ต้องใช้ SACK option)
• ใช้สำหรับ flow control และ error recovery
รับ seq=1001-2000 → ack=2001 (ต้องการ byte ลำดับ 2001)
Data Offset (Header Length) 4 bits
ระบุความยาวของ TCP header เป็นหน่วย 32-bit words (4 bytes) มีค่าขั้นต่ำ 5 (20 bytes) และสูงสุด 15 (60 bytes)
การคำนวณ:
• Header Length = Data Offset × 4 bytes
• ค่า 5 = header ขั้นต่ำ 20 bytes (ไม่มี options)
• ค่า 15 = header สูงสุด 60 bytes (40 bytes options)
• ใช้หา starting position ของ data
0x5 (5) = 20 bytes | 0xF (15) = 60 bytes
Reserved 4 bits
สำรองไว้สำหรับการใช้งานในอนาคต ปัจจุบันต้องเป็น 0 ทั้งหมด เดิมเคยมี 6 bits แต่ถูกย้ายไปเป็น ECN flags
ประวัติ:
• RFC 793: สำรอง 6 bits
• RFC 3168: ใช้ 2 bits สำหรับ ECN (CWR, ECE)
• ปัจจุบัน: เหลือ 4 bits ที่ยังไม่ใช้
• ต้องเป็น 0 เพื่อ backward compatibility
ค่าที่ถูกต้อง: 0x0 (0000 binary)

Window Size 16 bits
ระบุจำนวน bytes ที่ receiver สามารถรับได้ในขณะนี้ ใช้สำหรับ flow control เพื่อป้องกัน buffer overflow
Sliding Window:
• ขนาดสูงสุด: 65535 bytes (โดยไม่ใช้ window scaling)
• Window scaling option: ขยายได้ถึง 1GB
• Zero window: หยุดการส่งชั่วคราว
• Dynamic adjustment: เปลี่ยนตาม buffer availability
0x4000 (16384) = รับได้ 16KB | 0x0000 = หยุดส่ง
Checksum 16 bits
ตรวจสอบความถูกต้องของ TCP header, data และ pseudo header จาก IP คำนวณโดย sender และตรวจสอบโดย receiver
การคำนวณ:
• ใช้ one's complement sum
• รวม: TCP header + data + IP pseudo header
• Pseudo header: src IP, dst IP, protocol, TCP length
• ตั้งค่า checksum = 0 ก่อนคำนวณ
Pseudo header ป้องกัน IP address เปลี่ยนแปลงโดยไม่รู้ตัว
Urgent Pointer 16 bits
ชี้ไปยัง byte สุดท้ายของข้อมูล urgent เมื่อ URG flag เป็น 1 คำนวณเป็น offset จาก sequence number
Out-of-band Data:
• ไม่ใช่การส่งข้อมูลแยกต่างหาก
• เป็นการทำเครื่องหมายข้อมูลใน stream
• Application ต้อง handle urgent data เอง
• ใช้น้อย เนื่องจากปัญหา implementation
seq=1000, urgent_ptr=10 → urgent data ถึง byte 1010
Options 0-40 bytes
ส่วนเสริมที่ไม่บังคับ สามารถมีได้หลาย options และต้อง padding ให้ครบ 32-bit boundary
Options ที่สำคัญ:
• MSS (Maximum Segment Size): ขนาดข้อมูลสูงสุด
• Window Scale: ขยาย window size เกิน 64KB
• SACK (Selective ACK): ยืนยันการรับข้อมูลเฉพาะส่วน
• Timestamp: วัดเวลา RTT และ PAWS
• No-operation (NOP): padding alignment
MSS=1460, WScale=7, SACK_OK, Timestamp=12345678
